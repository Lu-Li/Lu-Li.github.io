Reconfigurable Cloud Computing Platform and MapReduce performance optimization

      I was impressed with the computing power of Hadoop during that previous project, but I also became aware of some of its performance limitations. Therefore, I pursued an opportunity to do research on reconfigurable cloud computing platforms at Tsinghua University’s Advanced Mobile Computing Technology center. This was an excellent opportunity because this cooperative project with Imperial College London provided me with an international vision and exposed me to other advanced scientific research.

      From my previous project, I knew that the current Hadoop implementation assumes that computing nodes in a cluster are homogeneous in nature, so its task scheduler has a slot-based coarse-grain resource allocator as what showed in picture 1 and ignored the performance bottlenecks or resource competitions which can obviously reduce the performance of Hadoop cluster. Hadoop resources can be classified into CPU, memory, disk I/O and network bandwidth. Hence MapReduce workloads can be classified into different categories according to the type of resource they mainly consume. If any resource reaches its bottleneck, it forces other resources to wait, and consequently reduce the overall performance. Therefore, balanced resource utilization as showed in picture 2 is the key to achieve high performance.

        
picture 1                                                                                                        picture 2

      For example, the map/reduce program that estimates Pi using monte-carlo method requires as much computation resource as possible leading to high CPU utilization with light disk I/O. Therefore, Pi program is typically CPU intensive, and CPU resource is the performance bottleneck. However, TeraSort, a map/reduce program that sorts the data into a total order shows heavy disk I/O and light CPU utilization which is totally different from Pi. So TeraSort is disk I/O intensive, and disk I/O is the bottleneck.

      Our project is to utilize reconfigurable processor on the high CPU utilization program to reduce the bottleneck and optimize MapReduce performance. The typical architecture of a reconfigurable processor is a main controller coupled with reconfigurable hardware, whose major feature is that the reconfigurable hardware can implement different functions.
